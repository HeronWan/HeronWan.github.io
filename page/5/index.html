2，图像运算

加法

img1 + img2 ,遵循规则：
a+b = (a+b) % 256,即结果对256取膜

cv2.add(img1,img2) ,遵循规则：
a+b = min(a+b,255), 即和大于255时结果取255

图像加权和

dst = cv2.addWeighted(src1,alpha,src2,beta,gamma) 参数不能省略
结果就是 dst = src * alpha +src2 *beta +gamma

加权和超过255，结果同样取255

按位逻辑运算

cv2.bitwise_and() 与
cv2.bitwise_or() 或
cv2.bitwise_xor() 异或
cv2.bitwise_not() 非

跟 0 位与结果为 0 ，变成黑色
跟255位与结果是本身 ，颜色不变

数字水印

最低有效位指一个二进制中的第0位，由于二进制图像处于载体图像的最低有效位上面，所以对于载体图像的影响非常不明显，具有较高的隐蔽性。

在必要时，我们可以直接将载体图像中的最低有效位提取出来，即可得到嵌入该位上的二值图像，达到提取秘密信息的目的。
这种信息隐藏也被称为数字水印，通过该方式可以实现信息影藏，版权认证，身份认证等功能

嵌入过程
  用数字水印信息将图像的最低有效位替换掉

提取过程
  提取出最低有效位的水印信息即可

1，图像处理基础

图像的表示

二值图像：仅仅包含黑色和白色两种颜色的图像，黑色表示为0，白色表示为1。

灰度图像：包含256个灰度级，数值区间[0,255]，其中 0 表示纯黑色， 255表示纯白色，中间数值表示纯白和纯黑之间的不同极度的灰度。

彩色图像： 有红绿蓝三个颜色的通道，能表现更丰富的细节信息，图像的通道顺序是RGB；特别注意，opencv中通道的顺序是 BGR，和图片通道顺序相反

    项目背景：去年我们导师接了一个公司的管理系统项目，这个系统主要是给某个审计单位对他的客户单位（某个买额公司）的交易合规性进行风险分析。飞检员从买药的会议现场收集相关的信息，然后输入得到后台，系统进行合规性判断，然后做风险分析。 我在这个项目中充当了一个小组长的身份，同时也负责了。

    ​ 前端用的是jsp。

    遇到的难点：一开始做出的页面加载特别的慢，然后，尝试一些方式进行优化：

    1. 增加服务器性能；

    2. 配置tomcat启动器的栈的内存

    3. 前端减少http的请求次数；

    4. 上传比较大的数据时候，反应很慢，然后浏览器会出现会弹出：将比较大的数据分成多份进行上传。效果很明显

前端笔记/weex

发表于 2020-07-18 更新于 2020-02-28 分类于 前端笔记
本文字数： 1.2k 阅读时长 ≈ 1 分钟
  1. weex环境搭建
    • 安装依赖node.js
    • npm install -g weex-toolkit
      • weex -v
    • weex init project_name
    • cd project_name
    • npm install
    • npm run serve
    • npm run dev
  2. weex通用样式
    1. 适配和缩略
      1. weex对于长度值目前只支持像素值，不支持相对单位(em/rem);适配以750px标准
      2. 设定边框，border目前不支持类似这样border：1px solid #ff0000;的这样组合写法
      3. 设定背景色，background目前不支持background:red的写法，需要改为具体的background-color：red。
    2. 其余需要注意的地方
      1. weex支持线性渐变linear-gradient不支持radial-gradient径向渐变
      2. weex中box-shadow仅仅支持IOS
      3. 目前组件无法定义一个或者几个角的border-radius,只对IOS有效，对andoid无效
      4. weex中，flexbox是默认并且唯一的布局模型，每个元素都默认拥有了display:flex属性
  3. 组件
    1. a组件定义了指向weex页面打包后的一个js地址
    2. 组件中无法添加文本，需要其中加上text组件才能添加文本
    3. 此组件支持除了自己外的所有weex组件作为子组件
    4. 支持所有通用样式
    5. 不要为a组件添加click事件
  4. 组件
    1. 组件用于在页面中嵌入一张网页；src用以资源地址。
    2. 不支持任何子组件
    3. pagestart，组件开始加载是执行
    4. pagefinish，组件完成加载时执行
    5. error,组件加载错误时执行
  5. webview组件
    1. 一系列组件操作接口，可以通过调用this.$refs.el 来获取元素的引用
    2. API：
      1. goback(webElement),加载历史记录里的前一个资源地址。
      2. goForward(webElement),加载历史记录里的下一个资源地址
      3. reload(webElement),刷新当前页面
  6. list组件 实现下拉刷新和上拉加载
    1. 组件是垂直列表功能的核心组件
    2. 组件，列表中的子项；类似于html中ul和li的关系
    3. 到达屏幕顶端时，吸附在屏幕顶端
    4. 给列表添加下拉刷新的功能
    5. loadmore事件，loadmoreoffset{number}触发事件所需距离
    6. scorll事件列表滚动式触发的该事件
    7. offset-accuracy{number}控制scroll事件发生的频率(默认10px)
  7. refresh组件
    1. 用于提供下拉加载功能。和的子组件，只有这两个包含时才能正常展示。
    2. refresh事件，当list组件和scroller组件被下拉时触发。
    3. pullingdown，当list组件和scroller组件被下拉时触发，可以从事件的参数对象中获取dy,pullingDistance,viewHeight,type;
  8. weex快速上手手册：weex.apache.org/cn/refrences
